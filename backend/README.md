# Backend веб-приложения для ведения личного бюджета

Данный каталог содержит серверную часть приложения, реализованную на **FastAPI** с использованием **PostgreSQL** в качестве базы данных.

## Используемые технологии

- Python 3
- FastAPI
- PostgreSQL
- asyncpg
- Docker
- Docker Compose
- JWT-аутентификация

## Структура проекта

```text
backend_with_compose/
├── app/
│   ├── api/                # Маршруты API
│   ├── core/               # Конфигурация, безопасность, зависимости
│   ├── database/           # Работа с БД, schema.sql, init_db.py
│   ├── models/             # Pydantic-схемы
│   ├── repositories/       # Запросы к базе данных
│   ├── services/           # Бизнес-логика
│   └── main.py             # Точка входа в приложение
├── .env                    # Переменные окружения
├── Dockerfile              # Сборка контейнера backend
├── docker-compose.yml      # Запуск backend и PostgreSQL
└── requirements.txt        # Зависимости Python
```

## Функциональность backend

Серверная часть поддерживает следующие функции:

- регистрация пользователя;
- вход в систему;
- получение и обновление данных пользователя;
- создание, просмотр, редактирование и удаление категорий;
- создание, просмотр, редактирование и удаление финансовых операций;
- планирование бюджета по категориям;
- сравнение плановых и фактических расходов.

## Подготовка к запуску

Перед запуском убедитесь, что у вас установлены:

- Docker
- Docker Compose

## Настройки окружения

Параметры подключения и настройки приложения находятся в файле `.env`.

Пример:

```env
POSTGRES_DB=budget_db
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
POSTGRES_HOST=db
POSTGRES_PORT=5432

BACKEND_PORT=8000

DATABASE_URL=postgresql://postgres:postgres@db:5432/budget_db
JWT_SECRET_KEY=change_me_to_a_long_random_secret_key
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=60
```

## Запуск проекта

Выполните команду:

```bash
docker compose up --build
```

После успешного запуска будут доступны:

- backend API: `http://localhost:8000`
- Swagger UI: `http://localhost:8000/docs`

## Остановка проекта

Для остановки контейнеров выполните:

```bash
docker compose down
```

## База данных

Структура базы данных задается в файле:

```text
app/database/schema.sql
```

Автоматическая инициализация базы данных выполняется через:

```text
app/database/init_db.py
```

В базе данных создаются следующие основные сущности:

- `users`
- `categories`
- `operations`
- `budget`

Также создается представление:

- `actual_expense_summary`

Оно используется для анализа фактических расходов по категориям.

## Полезные замечания

1. При первом запуске PostgreSQL будет создана база данных с параметрами из `.env`.
2. Для корректной работы backend важно, чтобы `DATABASE_URL` соответствовал параметрам подключения к БД.
3. Для учебного проекта рекомендуется сначала проверить работу API через Swagger, а затем подключать frontend.
